A super simple guide to get your projects online!

Step 1: What are Git and GitHub?

Think of Git as a magical tool that keeps track of all the changes you make to your code. It's like having an "undo" button for your entire project, and you can see who changed what and when.

GitHub is like a giant online storage space for your Git projects. It's where you can share your code with others, collaborate, and show off your work to the world!

Ready to get started?

Step 2: Install Git on Your Computer

Before we can use Git, you need to install it. It's free and easy!

Go to the official Git website and download the version for your operating system (Windows, macOS, Linux). Follow their installation instructions.

Download Git Here

Once installed, you might need to restart your computer or terminal.

Step 3: Open Your Terminal or Command Prompt

This is where you'll type Git commands. It's a text-based interface for your computer.

  • Windows: Search for "Command Prompt" or "PowerShell".
  • macOS: Search for "Terminal".
  • Linux: Look for "Terminal" in your applications.

Open it up! It will look like a black or white window with text.

Step 4: Navigate to Your Project Folder

You need to tell Git which folder your project is in. Use the `cd` (change directory) command.

Replace `your/project/path` with the actual path to your project folder (e.g., `C:\Users\YourName\Documents\MyWebsite` or `/Users/YourName/Documents/MyWebsite`).

cd your/project/path

Press Enter after typing the command.

Step 5: Initialize Git in Your Project

This command tells Git to start tracking changes in your current folder. You only do this once per project.

git init

You should see a message like "Initialized empty Git repository in..."

Step 6: Stage Your Files

Before Git saves your changes, you need to tell it which files you want to include in the next "snapshot" (commit). This is called "staging."

The `.` means "all files" in the current directory. If you only want to add specific files, you can replace `.` with `filename.html` or `folder/another-file.js`.

git add .

Press Enter. Nothing might appear, but Git has prepared your files.

Step 7: Commit Your Changes

Now, let's save a snapshot of your staged files. This is called a "commit." Every commit needs a message explaining what changes you made.

The `-m` stands for "message." Make your message short and descriptive!

git commit -m "First commit"

Press Enter. You'll see details about the files committed.

Step 8: Create a New Repository on GitHub

Now, let's make a home for your project on GitHub!

  1. Go to github.com and sign in (or create an account).
  2. On the left sidebar, click the green "New" button (or the "+" icon in the top right and select "New repository").
  3. Repository name: Type a name for your project (e.g., `my-first-website`).
  4. Description (optional): Add a short description.
  5. Choose Public or Private. Public means anyone can see it; Private means only you and people you invite can see it.
  6. IMPORTANT: Do NOT check "Add a README file", "Add .gitignore", or "Choose a license" at this stage. We'll add our own files.
  7. Click the green "Create repository" button.

After creating, you'll see a page with instructions. Look for the section that says "â€¦or push an existing repository from the command line". We'll use the next two commands from there.

Step 9: Connect Your Local Project to GitHub

This command tells your local Git project where its online home (the GitHub repository) is. You'll find this command on the GitHub page you just created.

Replace `` with the actual URL provided by GitHub (it usually looks like `https://github.com/YourUsername/your-repo-name.git`).

git remote add origin <YOUR_REPOSITORY_URL>

Press Enter. `origin` is just a common nickname for your main remote repository.

Step 10: Push Your Code to GitHub!

This is the big moment! This command sends all your committed changes from your computer to your GitHub repository.

  • `push` means send.
  • `-u` sets up a connection so you don't have to type `origin main` every time you push.
  • `origin` is the nickname for your GitHub repo.
  • `main` (or `master`) is the name of your main branch.
git push -u origin main

Press Enter. If it's your first time, you might be asked to log in to GitHub.

Step 11: Verify on GitHub

Go back to your GitHub repository page in your web browser. Refresh the page.

You should now see all your files and folders from your project! Congratulations, you've successfully pushed your code to GitHub!

Your project is now online and version-controlled!

Step 12: What's Next? (More Baby Steps)

You've mastered the basics! Here are a few more useful commands:

Check Status:

See what files have been changed, added, or are untracked.

git status

Pull Changes:

If you're working with others, or on another computer, use this to get the latest changes from GitHub.

git pull origin main

Repeat the Cycle:

Whenever you make changes to your code, repeat these steps:

  1. git add . (Stage your new changes)
  2. git commit -m "Your descriptive message" (Save the snapshot)
  3. git push origin main (Send to GitHub)

Keep practicing, and you'll be a Git pro in no time!
